# Rubric for Mini Project 1 - Data Wrangling

## Highest Possible Score - 215 points

### 1 Adherence to project requirements [40 pts]

1. Data acquisition through web scraping and web API for **three different sources** (at least one source from web scraping, then at least one source from web APIs) - 5 pts
2. Data has been stored in a **proper data format**. Should have at least one structured data format (other than SQL database), one semi-structured data format, and one SQL data file. - 5 pts
3. A data analysis pipeline which can be a exploratory data analysis, descriptive analysis, or prescriptive analysis has been made from the extracted data - 5 pts
4. Python Environment has been properly defined as an `environment.yml` file with specified library versions for maximum reproducibility - 5 pts
5. Git branch `develop` has been properly created properly and all relevant changes have been pushed to this branch - 5 pts
6. All code used for data extraction and data analysis are placed inside modules and/or python packages, only visualization and final analysis are contained in the `report.ipynb` notebook - 5 pts
7. Unit tests have been properly created for all functions and pipelines of each module. A text or image file showing test result have been show in the repository. - 5 pts
8. Submission has been made thru creation of a pull request from the branch `develop` to `main` - 5 pts

### 2 Mastery of data wrangling techniques [45 pts]

1. Use of appropriate data formats, proper parsing, reading, and loading of data files have been accomplished - 10 pts
2. Techniques of regular expression has been integrated meaningfully into the data wrangling pipeline - 10 pts
3. Scripts involving creation, modification, and retrieval of SQL tables are integrated into the data wrangling and analysis pipeline - 15 pts
4. Web scraping and Web API methodologies adhere to the best practices and standard discussed in class - 10 pts

### 3 Code organization and hygiene [15 pts]

1. All written code are PEP8 compliant with proper documentation - 5 pts
2. Variables, modules, and functions are named consistently and intelligibly - 5 pts
3. Code shows clear modularization with minimal redundancy - 5 pts

### 4 Quality of analysis and insights [20 pts]

1. Design constraints and choices are intelligible and extensive and properly articulated - 10 pts
2. A clear business value of the extracted insights have been articulated - 10 pts

### 5 Creativity and novelty [20 pts]

1. A degree of algorithmic and/or analytic novelty is presented by the work - 10 pts
2. Shows originality and creativity in how different techniques are applied and integrated to solve the task - 10 pts

### 6 Team workflow and efficiency [15 pts]

1. Work is appropriately and equitably distributed among team members, with clear ownership and accountability - 5 pts
2. Commits are atomic, logically scoped, and accompanied by clear, descriptive commit messages - 5 pts
3. Tasks are completed in a timely manner - 5 pts

### 7 Completeness of the report [45 pts]

1. Title succinctly reflects what the study is about - 5 pts
2. Executive summary provides a complete overview of the study - 5 pts
3. Context problem has been defined properly in the introduction - 5 pts
4. Business value has been properly articulated - 5 pts
5. Description of the data sources and extraction are present - 5 pts
6. Methodology has been articulated and all design constraints have been provided - 5 pts
7. Each insight gleaned from each analysis have been discussed - 5 pts
8. A summary of key findings and takeaway message has been provided in the conclusion - 5 pts
9. Actionable suggestions has been given in the recommendation section - 5 pts

### 8 Presentation and aesthetics [15 pts]

1. Visualizations are creative, professional, and strongly integrates with the text discussion - 5 pts
2. Report is well-written; There are minimal grammar and spelling errors. Text is easy to digest - 5 pts
3. All visual elements have proper labels, self-contained captions, and is referred properly in the text. - 5 pts
