# Mini Project 1: Data Wrangling

This repository contains the starter code for **Mini Project 1** in **DMW 2301 (Data Mining and Wrangling)** for BSDSBA 2028.

## Project Goal 🎯

The objective of this mini-project is to apply the data wrangling skills you've learned in the first half of the course.

For this mini-project, you are expected to:

1. **Acquire data** through web scraping and usage of web APIs.
2. **Store the collected data** in an appropriate format (e.g., `CSV`, `JSON`, `SQL`, etc.)
3. **Perform an exploratory data analysis (EDA)** or any simple data and/or statistical analysis.
4. **Generate insights** with clear business value.

Your output should demonstrate not only technical correctness, but also meaningful interpretation of the analysis of the data you've obtained.

## Project Themes 💡

You may choose one of the suggested topics below or propose your own (subject to approval):

1. **Travel Companion Application** ✈️
    - Extract and store data from APIs (e.g., Places API), Wikipedia, or other travel-related.
    - Build a dataset of tourist attractions, restaurants, shopping districts, etc., for a specific location.
    - Perform data analysis to generate travel insights or recommendations for tourists.
2. **Trader Companion Application** 📈
    - Extract stock market data from APIs (e.g., Alpha Vantage).
    - Complement the market data by scraping relevant financial news articles.
    - Perform analysis to generate buy/hold/sell recommendations supported by both quantitative data and contextual information.
3. **Custom Project** 🔧
    - You may also pursue any other use-case that: (1) extracts data via API and web scraping, (2) stores the data in an appropriate formate, and (3) performs analysis that generates actionable business insights.

Creativity is strongly encouraged, but ensure that you adhere to the standards and basic guidelines of the data wrangling techniques we've discussed in class.

## Setup Instructions and Submission Guidelines 🛠

1. **Repository Setup**
    - Clone the repository to your local machine.
    - Install the recommended environment using the provided `environment.yml`
    - If you use additional libraries, update the `environment.yml` file accordingly.
2. **Git Branching**
    - Create a branch named `develop`.
    - All changes must be committed and pushed to this branch.
    - You may create additional feature branches if necessary.
3. **Code Organization**
    - Place all code used in data extraction and data analysis inside properly named **modules** or **packages**.
    - The `report.ipynb` notebook should only contain final analysis, visualizations, and explanations.
    - Write unit tests for each function in your modules.
4. **Report Requirements**
    - Write your final report in the `report.ipynb`.
    - The report should include:
         - An executive summary of your work
         - Problem definition and business value
         - Data description, cleaning, and transformation steps
         - Methodology for data wrangling and processing
         - Data analysis including visualizations to support your analysis
         - Key findings and business insights
         - Recommendation for extension of your work and clarification of limitations
5. **Submission**
     - Once completed, create a **Pull Request (PR)** from `develop` to `main`.
     - Include a concise but clear summary in your pull request describing:
         - The project objective
         - The data sources used
         - Key findings
         - Major design decisions
         - Contributions of each team member
     - Your project will be graded based on the contents of this pull request.

## Grading Criteria 📊

Projects will be evaluated based on:
- Adherence to project requirements
- Mastery of data wrangling techniques
- Code organization and hygiene
- Quality of analysis and insights
- Creativity and novelty
